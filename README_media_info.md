# media_info.py

`media_info.py` — это Python-скрипт для подробного анализа медиафайлов. Он читает данные файловой системы, EXIF-метаданные изображений, GPS-координаты, параметры съёмки, а также дополнительные медиаданные для фото, видео и аудио.

Скрипт подходит для:

- быстрой проверки, что реально содержится в файле
- просмотра EXIF и параметров съёмки
- извлечения GPS-координат и места съёмки
- анализа фото, видео и аудио из одной утилиты
- вывода информации в консоль или в JSON для дальнейшей обработки

---

## Возможности

Скрипт умеет извлекать:

### 1. Данные файловой системы

- имя файла
- путь к файлу
- расширение
- размер в байтах и в удобочитаемом виде
- MIME-тип
- дату создания
- дату изменения
- дату последнего доступа

### 2. Общую информацию о медиафайле

- тип медиа: image / video / audio / unknown
- формат изображения
- цветовой режим
- ширину и высоту
- количество кадров для анимированных изображений
- прозрачность

### 3. EXIF и параметры съёмки для фото

- производитель камеры
- модель камеры
- модель объектива
- ПО, которым обрабатывался файл
- автор / copyright
- дата и время съёмки
- ISO
- выдержка
- диафрагма
- фокусное расстояние
- ориентация
- вспышка
- баланс белого
- цветовое пространство
- цифровой зум
- режим экспозиции и замера

### 4. GPS-данные

- широта
- долгота
- высота
- GPS date stamp / time stamp
- исходные GPS-поля EXIF

### 5. Дополнительные медиаданные

Для видео и аудио, а также иногда для изображений:

- контейнер
- кодек
- длительность
- bitrate
- frame rate
- параметры потоков
- дополнительные служебные данные треков

### 6. Обратное геокодирование

Если в файле есть GPS и включён соответствующий флаг, скрипт может попытаться определить текстовый адрес по координатам через OpenStreetMap Nominatim.

## Требования

- Python 3.10+

Рекомендуемые зависимости:

```bash
pip install Pillow pymediainfo
```

Опционально для HEIC/HEIF:

```bash
pip install pillow-heif
```

Опционально для более широкого анализа видео/аудио:

- установленный `ffprobe` из FFmpeg

Проверка:

```bash
ffprobe -version
```

## Запуск

### Базовый запуск

```bash
python media_info.py "C:\media\photo.jpg"
```

### С определением адреса по GPS

```bash
python media_info.py "C:\media\photo.jpg" --reverse-gps
```

### Вывод в JSON

```bash
python media_info.py "C:\media\video.mp4" --json
```

---

## Аргументы командной строки

```text
media_info.py [-h] [--reverse-gps] [--json] file
```

### Позиционные аргументы

- `file` — путь к медиафайлу

### Опции

- `-h, --help` — показать справку
- `--reverse-gps` — попытаться определить адрес по GPS через OpenStreetMap Nominatim
- `--json` — вывести результат в формате JSON

## Особенности и ограничения

### 1. Не все файлы содержат EXIF

Некоторые источники удаляют метаданные:

- мессенджеры
- соцсети
- редакторы изображений
- экспортированные или пересохранённые файлы

### 2. GPS может отсутствовать

Даже если файл снят на телефон, GPS-координаты могут быть отключены в настройках камеры или вырезаны при экспорте.

### 3. Дата файла и дата съёмки — не одно и то же

- дата файловой системы — когда файл был создан/скопирован/изменён на диске
- дата EXIF — когда произошла съёмка

Эти значения часто различаются.

### 4. Для видео больше информации даёт MediaInfo или ffprobe

EXIF полезен в основном для изображений. Для видео и аудио лучше работает `pymediainfo` или `ffprobe`.

### 5. HEIC/HEIF требуют дополнительной поддержки

Для некоторых файлов iPhone понадобится `pillow-heif`.

## Диагностика проблем

### Ошибка `Pillow не установлен`

Установите зависимость:

```bash
pip install Pillow
```

### Нет данных по видео

Установите одно из:

```bash
pip install pymediainfo
```

или FFmpeg / ffprobe.

### HEIC не открывается

Установите:

```bash
pip install pillow-heif
```

### GPS-адрес не определяется

Проверьте:

- есть ли в файле GPS
- есть ли доступ к интернету
- не блокируется ли запрос к OpenStreetMap Nominatim
